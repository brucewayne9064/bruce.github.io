Hexo常用命令

发表于2023-02-28|更新于2024-04-13|博客搭建
|字数总计:496|阅读时长:1分钟|阅读量:

hexo是一个可以搭建在GITHUB page上的博客框架，它的文档网站是https://hexo.io/zh-cn/

github仓库是https://github.com/hexojs/hexo

本文主要介绍在使用该框架时常用到的命令

命令

new

该命令用于新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

参数 描述
-p, --path 自定义新文章的路径
-r, --replace 如果存在同名文章，将其替换
-s, --slug 文章的 Slug，作为新文章的文件名和发布后的 URL

generate

修改完配置需要生成静态文件

选项 描述
-d, --deploy 文件生成后立即部署网站
-w, --watch 监视文件变动
-b, --bail 生成过程中如果发生任何未处理的异常则抛出异常
-f, --force 强制重新生成文件 Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。 使用该参数的效果接近 hexo clean && hexo generate
-c, --concurrency 最大同时生成文件的数量，默认无限制

server

启动服务器。默认情况下，访问网址为： http://localhost:4000/。本地预览的时候使用

选项 描述
-p, --port 重设端口
-s, --static 只使用静态文件
-l, --log 启动日记记录，使用覆盖记录格式

clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

deploy

部署网站，每次先clean，再generate，再deploy，仓库里存的内容是blog文件夹中的public文件夹的内容

参数 描述
-g, --generate 部署之前预先生成静态文件
